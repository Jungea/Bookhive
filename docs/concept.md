# 독서 이력 기반 도서관 타이쿤 서비스 기획 정리

## 서비스 핵심
읽은 만큼, 남긴 만큼 도서관이 자라는 독서 기록 서비스.
독서 기록이 단순 리스트가 아니라 실제 도서관 재고/손님/성장 요소로 반영된다.

핵심 컨셉:
- 내 취향이 공간이 된다.
- 완독하지 않아도 부분 읽기/기록으로 도서관에 반영 가능.
- 독서 기록 = 도서관 운영 자원.

---

## 독서 콘텐츠 종류

### 일반 책
- ISBN 기반 등록
- 기본 완독 등록
- 페이지 기반
- 두께 UI 반영
- 대여 기간 차등

### 웹소설 / 연재작
- 화수 기반
- 최신화 개념
- 열람석 운영
- 손님이 앉아서 읽음
- N명 동시 열람 가능
- 단골 손님 생성

### 비출간 작품
- 개인 등록 가능
- 인디 연재작
- 팬 창작물
- 커뮤니티 작품

### 직접 쓴 글
- 추후 검토
- 소설 / 에세이 / 메모

---

## 도서관 등록 규칙

### 기본 등록
- 첫 완독 시 재고 1권 생성

### 부분 읽기 등록
- 완독하지 않아도 일부 읽기/기록 기반으로 도서관 반영 가능
- 서비스 철학 유지 포인트

### 재고 증가
1. 첫 완독 → 1개
2. 대여 횟수 누적 → 호감도 상승 → 가득 차면 재고 +1
3. 독후감 작성 / 재독 → 추가 재고 보너스

---

## 독서 행동 → 게임 반영

### 읽는 중
- 해당 분야 손님 홍보 효과
- 특정 장르 방문객 증가
- 책상/카트/책갈피로 시각 표현

### 완독
- 재고 등록
- 대여 가능
- 메인 성장 루프

### 독후감
- BEST 영역 전시
- 취향 무관 손님도 대여 확률 증가

### 재독
- 추가 재고 및 대표 소장본 후보

### 연재작
- 열람석 콘텐츠
- 단골 생성
- 최신화 확인 접속 유도

---

## 손님 시스템

손님 속성:
- 취향
- 요구 장르
- 성격
- 만족도
- 단골 여부

### 네임드 단골
예:
- 교수 → 서울대 김교수
- 이름/개별 기록 존재
- 특별 UI

### 퀘스트
손님이 특정 책 요청
예:
- 철학 입문서 찾음
- 특정 작가 요청
- 특정 장르 요청

---

## 운영 요소

### 반납 시스템
- 책 대여 후 반납 시 사서 카드 더미 생성
- 유저가 클릭해 정리해야 재대여 가능

### 반납 이벤트
- 정상 반납
- 연체
- 훼손
- 메모 발견

### 운영 행동
- 정리
- 우선 배치
- 요청 처리
- 서가 확장

---

## 성장 자원

### 평판
누적 지표
- 절대 감소 없음
- 해금 기준
- 특수 손님
- 구역 확장 조건

### 후원금
소비 재화
- 책장 구매
- 열람실 추가
- 구역 확장
- 시설 업그레이드

### 일일 정산
- 오늘 발생한 후원금은 당일 수령
- 미수령 시 누락/감소 검토
- 접속 동기 제공
