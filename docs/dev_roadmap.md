# 개발 진행 순서

## 1. 게임 기본 씬 구축
- 바닥
- 벽
- 책장
- 책
- 사서데스크
- 사서
- 손님
- Phaser 렌더링 우선

## 2. 독서 이력 CRUD
- 책 등록
- 메모
- 독후감
- 읽기 상태
- 재독

## 3. 완독 연동
- 완독 시 책장 노출
- 책 두께 반영
- 슬롯 배치

## 4. 대여 시스템
- 손님 입장
- 책 탐색
- 대여
- 반납 카드 생성

## 5. 손님 요청 퀘스트
- 특정 장르/책 요청
- 완료 시 평판/후원금 지급

## 6. 평판 / 후원금
- 일일 정산
- 수령 시스템
- 성장 자원 소비

## 7. 반납 이벤트
- 연체
- 훼손
- 메모 발견

## 8. 연재작 열람석
- 좌석 점유
- 단골 생성
- 최신화 체크

## 9. 단골 시스템
- 네임드 손님
- 취향 진화

## 10. 도서관 확장
- 책장 확장
- 열람실 확장
- 특별 구역

## 구조 원칙
- React: 상태/UI
- Supabase: 데이터 저장
- Phaser: 화면 렌더링
- 비즈니스 로직은 React 중심
